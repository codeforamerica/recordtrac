{% trans %}Best Practices to Consider{% endtrans %}

{% trans %}Content should only be removed or edited if sensitive or confidential information is revealed. If this happens, you should:{% endtrans %}

  • {% trans %}Save a copy of the original message. (This will have to be done outside of RecordTrac. There is no way to hide a message from public view.){% endtrans %}
  • {% trans %}Edit the message to indicate why it needs to be removed.{% endtrans %}
  • {% trans %}Notify the requester why their post or answer was removed.{% endtrans %}
  • {% trans %}Provide guidance to the requester on how they can get the record they need.{% endtrans %}

{% trans %}If a city staff member enters information incorrectly, simply add a note explaining the mistake.{% endtrans %}

{% trans %}If a member of the public enters incorrect information, the requester (or a city staff member) can add a note correcting the mistake.{% endtrans %}

{% trans %}Sometimes it’s necessary to create a new request. If a new request must be created, we suggest you do the following:{% endtrans %}

  • {% trans %}Create a new request with the proper information.{% endtrans %}
  • {% trans %}In the old request, include a note explaining what is wrong with it and a link to the new request.{% endtrans %}
  • {% trans %}Close out the old request.{% endtrans %}
  • {% trans %}In the new request, reference and/or provide a link to the old request.{% endtrans %}

{% trans %}Although RecordTrac has a spam filter, it may receive spam. When confronted with spam, close the request with a note indicating why it is not a public records request.{% endtrans %}
